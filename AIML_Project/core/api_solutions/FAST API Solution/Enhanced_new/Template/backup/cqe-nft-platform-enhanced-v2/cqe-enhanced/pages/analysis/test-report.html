CRITICAL: T03_Payment — P95 14.82s (SLA: 5s, +196%) P0
Root Cause: Oracle lock contention (ORA-00054) on PAYMENT_TXN table during concurrent writes from 2,847 Vusers. AppDynamics node pay-proc-02: 91% CPU, 4.7% error rate.
Evidence: AWR: "enq: TX - row lock contention" at 20.32ms avg. Splunk: 2,841 ORA-00054 errors. Kibana: /payment P95 = 3,421ms.
Fix: Implement optimistic locking. Add composite index on PAYMENT_TXN(STATUS, CREATED_DATE). Increase JDBC pool size 50→150.
HIGH: T05_Credit_Limit — 90.9% Pass Rate (SLA: 99%) P1
